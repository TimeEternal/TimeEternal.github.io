## 第一部分：大语言模型 (LLM) 是什么？

### 1.1 一句话定义

> **大语言模型（LLM）是一个读过几乎所有互联网文本的「超级预测机」，它通过预测下一个词来生成连贯的回答。**

### 1.2 工作原理图解

```
┌─────────────────────────────────────────────────────────────┐
│                    大语言模型工作流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   输入: "今天天气"                                           │
│         ↓                                                   │
│   ┌─────────────────────────────────────┐                   │
│   │     Transformer 模型处理            │                   │
│   │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐   │                   │
│   │  │今天 │ │天气 │ │怎么样│ │ ?   │   │                   │
│   │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘   │                   │
│   │     ↓       ↓       ↓       ↓      │                   │
│   │  [Embedding 向量化]                 │                   │
│   │     ↓       ↓       ↓       ↓      │                   │
│   │  [Attention 注意力机制]             │                   │
│   │     ↓       ↓       ↓       ↓      │                   │
│   │  [Feed Forward 前馈网络]            │                   │
│   │     ↓                              │                   │
│   └─────┼─────────────────────────────┘                   │
│         ↓                                                   │
│   输出: "很好，阳光明媚"                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心概念解释

| 概念 | 通俗解释 | 技术定义 |
|------|---------|---------|
| **Transformer** | 模型的「大脑架构」 | 一种基于注意力机制的神经网络架构 |
| **Token** | 文本的「最小单位」 | 一个 Token ≈ 0.75 个英文单词 或 0.5 个汉字 |
| **Embedding** | 文字的「数字表示」 | 把文字转换成向量（一串数字） |
| **Attention** | 「关联度计算」 | 决定哪些词之间有联系 |
| **Context Window** | 「短期记忆容量」 | 模型一次能处理的文本长度 |

### 1.4 主流 LLM 对比

```
┌──────────────┬─────────────┬─────────────┬─────────────┐
│    模型       │   公司       │   特点       │   上下文    │
├──────────────┼─────────────┼─────────────┼─────────────┤
│ Claude 4     │ Anthropic   │ 安全、长文   │ 200K tokens │
│ GPT-4o       │ OpenAI      │ 多模态、快   │ 128K tokens │
│ Gemini 2     │ Google      │ 原生多模态   │ 1M tokens   │
│ Llama 4      │ Meta        │ 开源、可定制 │ 128K tokens │
└──────────────┴─────────────┴─────────────┴─────────────┘
```

## 第二部分：Agent —— 从「聊天」到「行动」

### 2.1 什么是 Agent？

> **Agent = LLM + 工具 + 自主决策能力**

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│     普通聊天机器人                    AI Agent             │
│    ┌───────────────┐              ┌───────────────┐        │
│    │               │              │               │        │
│    │  只能对话     │              │  能执行任务   │        │
│    │  不能操作     │      →       │  能调用工具   │        │
│    │  被动响应     │              │  主动规划     │        │
│    │               │              │               │        │
│    └───────────────┘              └───────────────┘        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 2.2 Agent 核心架构

```
                    ┌─────────────────────────────┐
                    │         用户请求             │
                    │   "帮我订明天去上海的机票"    │
                    └──────────────┬──────────────┘
                                   ↓
                    ┌─────────────────────────────┐
                    │       Agent 大脑 (LLM)       │
                    │                             │
                    │  1. 理解意图：订机票          │
                    │  2. 规划步骤：               │
                    │     - 查询航班              │
                    │     - 比较价格              │
                    │     - 完成预订              │
                    │  3. 决策：调用哪些工具        │
                    └──────────────┬──────────────┘
                                   ↓
         ┌─────────────────────────┼─────────────────────────┐
         ↓                         ↓                         ↓
    ┌─────────┐              ┌─────────┐              ┌─────────┐
    │ 航班查询 │              │ 价格比较 │              │ 支付工具 │
    │  Tool   │              │  Tool   │              │  Tool   │
    └────┬────┘              └────┬────┘              └────┬────┘
         ↓                         ↓                         ↓
    ┌─────────┐              ┌─────────┐              ┌─────────┐
    │ 航班API  │              │ 比价网站 │              │ 支付API  │
    └─────────┘              └─────────┘              └─────────┘
```

### 2.3 Agent 的关键能力

| 能力 | 说明 | 示例 |
|------|------|------|
| **感知** | 理解环境和输入 | 解析用户的自然语言请求 |
| **推理** | 逻辑分析和决策 | 判断需要调用哪些工具 |
| **行动** | 执行具体操作 | 调用 API、读写文件 |
| **记忆** | 保存上下文信息 | 记住用户偏好和历史对话 |
| **学习** | 从反馈中改进 | 用户纠正后记住正确做法 |

## 第三部分：Tool Calling —— 工具调用的魔法

### 3.1 什么是 Tool Calling？

> **Tool Calling 让 LLM 能够「操作」外部世界，突破纯文本的限制。**

### 3.2 工作流程详解

```
┌─────────────────────────────────────────────────────────────────┐
│                     Tool Calling 完整流程                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  步骤1: 用户请求                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 用户: "北京今天天气怎么样？"                               │    │
│  └─────────────────────────────────────────────────────────┘    │
│                            ↓                                    │
│  步骤2: 模型分析                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ LLM: 我需要调用 get_weather 工具                          │    │
│  │      参数: city = "北京"                                  │    │
│  └─────────────────────────────────────────────────────────┘    │
│                            ↓                                    │
│  步骤3: 返回工具调用请求                                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ {                                                        │    │
│  │   "name": "get_weather",                                 │    │
│  │   "arguments": { "city": "北京" }                         │    │
│  │ }                                                        │    │
│  └─────────────────────────────────────────────────────────┘    │
│                            ↓                                    │
│  步骤4: 系统执行工具                                              │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 系统调用天气 API，获取结果                                 │    │
│  │ 结果: { "temp": 25, "weather": "晴", "humidity": 45 }    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                            ↓                                    │
│  步骤5: 将结果返回给模型                                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ 工具结果 → LLM                                            │    │
│  └─────────────────────────────────────────────────────────┘    │
│                            ↓                                    │
│  步骤6: 模型生成最终回答                                          │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ "北京今天天气晴朗，气温25度，湿度45%，适合外出活动。"       │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 工具定义格式

```json
{
  "name": "get_weather",
  "description": "获取指定城市的天气信息",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "城市名称，如：北京、上海"
      }
    },
    "required": ["city"]
  }
}
```

### 3.4 消息格式详解

```json
// 1. 用户消息
{
  "role": "user",
  "content": "北京今天天气怎么样？"
}

// 2. 模型决定调用工具
{
  "role": "assistant",
  "content": null,
  "tool_calls": [{
    "id": "call_abc123",
    "type": "function",
    "function": {
      "name": "get_weather",
      "arguments": "{\"city\": \"北京\"}"
    }
  }]
}

// 3. 工具执行结果
{
  "role": "tool",
  "tool_call_id": "call_abc123",
  "content": "{\"temp\": 25, \"weather\": \"晴\"}"
}

// 4. 模型最终回答
{
  "role": "assistant",
  "content": "北京今天天气晴朗，气温25度。"
}
```

## 第四部分：MCP —— 模型上下文协议

### 4.1 MCP 是什么？

> **MCP (Model Context Protocol) 是一个开放标准，让 AI 应用和外部数据源/工具之间建立统一的连接方式。**

```
┌─────────────────────────────────────────────────────────────────┐
│                         MCP 架构图                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                    ┌─────────────────┐                          │
│                    │   MCP Client    │                          │
│                    │  (Claude/应用)   │                          │
│                    └────────┬────────┘                          │
│                             │                                   │
│                    ┌────────┴────────┐                          │
│                    │   MCP Server    │                          │
│                    │   (协议层)       │                          │
│                    └────────┬────────┘                          │
│                             │                                   │
│         ┌───────────────────┼───────────────────┐               │
│         ↓                   ↓                   ↓               │
│   ┌───────────┐       ┌───────────┐       ┌───────────┐        │
│   │ 文件系统   │       │  数据库   │       │   API    │        │
│   │  MCP Host │       │ MCP Host  │       │ MCP Host │        │
│   └───────────┘       └───────────┘       └───────────┘        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 MCP 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **MCP Server** | 提供工具/资源的服务端 | USB 设备 |
| **MCP Client** | 使用工具的应用端 | 电脑 |
| **Resources** | 可读取的数据 | 文件、数据库记录 |
| **Tools** | 可执行的函数 | API 调用 |
| **Prompts** | 预定义的提示模板 | 快捷指令 |

### 4.3 MCP vs 传统集成

```
传统方式（为每个工具写适配器）:
┌─────────┐     ┌─────────┐     ┌─────────┐
│ Claude  │────→│ 适配器A │────→│ 工具 A  │
│  应用   │────→│ 适配器B │────→│ 工具 B  │
│         │────→│ 适配器C │────→│ 工具 C  │
└─────────┘     └─────────┘     └─────────┘
    问题：N 个工具需要 N 个适配器，维护成本高

MCP 方式（统一协议）:
┌─────────┐     ┌─────────┐     ┌─────────┐
│ Claude  │     │         │────→│ 工具 A  │
│  应用   │────→│   MCP   │────→│ 工具 B  │
│         │     │ (统一)  │────→│ 工具 C  │
└─────────┘     └─────────┘     └─────────┘
    优势：一次接入，处处可用
```

## 第五部分：Skill —— Agent 的技能包

> **Skill 是一组预定义的能力包，包含工具、提示词、工作流，让 Agent 能够完成特定领域的任务。**

```
┌─────────────────────────────────────────────────────────────────┐
│                        Skill 结构示意                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                      Skill: 天气助手                      │  │
│   │  ┌───────────────────────────────────────────────────┐  │  │
│   │  │ SKILL.md (技能描述)                                │  │  │
│   │  │ - 技能名称、用途说明                               │  │  │
│   │  │ - 触发条件                                         │  │  │

│   │  ┌───────────────────────────────────────────────────┐  │  │
│   │  │ tools/ (工具定义)                                  │  │  │
│   │  │ - get_weather.py                                  │  │  │
│   │  │ - get_forecast.py                                 │  │  │
│   │  └───────────────────────────────────────────────────┘  │  │
│   │  ┌───────────────────────────────────────────────────┐  │  │
│   │  │ references/ (参考文档)                             │  │  │
│   │  │ - api_docs.md                                     │  │  │

│   │  └───────────────────────────────────────────────────┘  │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 Skill 与 Tool 的区别

| 维度 | Tool | Skill |
|------|------|-------|
| 粒度 | 单个函数 | 功能集合 |

## 第六部分：Session —— 对话的容器

### 6.1 什么是 Session？

> **Session 是一次完整对话的生命周期容器，保存了对话历史、上下文状态和用户信息。**

```
┌─────────────────────────────────────────────────────────────────┐
│                      Session 生命周期                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   创建 Session                                                  │
│       ↓                                                         │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                     Session                              │  │
│   │  ┌─────────────────────────────────────────────────┐    │  │
│   │  │ 消息历史                                          │    │  │
│   │  │ [User] 你好                                      │    │  │
│   │  │ [Assistant] 你好！有什么可以帮你的？              │    │  │
│   │  │ [User] 帮我查一下天气                            │    │  │
│   │  │ [Tool Call] get_weather("北京")                  │    │  │
│   │  │ [Tool Result] {"temp": 25, "weather": "晴"}      │    │  │
│   │  │ [Assistant] 北京今天天气晴朗...                   │    │  │
│   │  └─────────────────────────────────────────────────┘    │  │
│   │  ┌─────────────────────────────────────────────────┐    │  │
│   │  │ 状态信息                                          │    │  │
│   │  │ - session_id: "sess_abc123"                      │    │  │
│   │  │ - user_id: "user_456"                            │    │  │
│   │  │ - created_at: 2026-03-20T10:00:00Z               │    │  │
│   │  │ - model: "claude-4"                              │    │  │
│   │  └─────────────────────────────────────────────────┘    │  │
│   └─────────────────────────────────────────────────────────┘  │
│       ↓                                                         │
│   结束/归档 Session                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 Session 类型

| 类型 | 说明 | 生命周期 |
|------|------|---------|
| **Main Session** | 主对话会话 | 持久保存 |
| **Sub-agent Session** | 子代理会话 | 任务完成后结束 |
| **Thread Session** | 线程会话 | 关联到特定话题 |

│   ┌─────────────────────────────────────────────────────────┐  │
│   │                    Gateway 核心                          │  │
│   │  ┌───────────────────────────────────────────────────┐  │  │

### 9.1 与大模型通信的完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│              用户与 LLM 的通信全链路                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① 用户输入                                                     │
│     "帮我分析一下这个代码文件"                                   │
│                     ↓                                           │
│  ② 客户端/平台接收                                               │
│     (Telegram/Discord/WebChat)                                  │
│                     ↓                                           │
│  ③ Gateway 处理                                                 │
│     ├── 识别用户身份                                            │
│     ├── 加载用户配置                                            │
│     ├── 构建系统提示词                                          │
│     └── 添加可用工具定义                                        │
│                     ↓                                           │
│  ④ 构建 API 请求                                                 │
│     ┌─────────────────────────────────────────────────────┐    │
│     │ POST /v1/messages                                    │    │
│     │ {                                                   │    │
│     │   "model": "claude-4",                              │    │
│     │   "max_tokens": 4096,                               │    │
│     │   "system": "你是一个有用的AI助手...",               │    │
│     │   "messages": [                                      │    │
│     │     {"role": "user", "content": "帮我分析..."}       │    │
│     │   ],                                                 │    │
│     │   "tools": [...]                                     │    │
│     │ }                                                   │    │
│     └─────────────────────────────────────────────────────┘    │
│                     ↓                                           │
│  ⑤ 发送到 LLM API                                               │
│     (Anthropic/OpenAI/Google)                                   │
│                     ↓                                           │
│  ⑥ LLM 处理并返回响应                                           │
│                     ↓                                           │
│  ⑦ 客户端处理响应                                               │
│     ├── 直接回答 → 显示给用户                                   │
│     └── 工具调用 → 执行工具 → 返回结果给LLM → 循环              │
│                     ↓                                           │
│  ⑧ 最终响应                                                     │
│     "这个代码文件实现了..."                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```json

    }
  ],
  "tools": [

      "name": "execute_code",
      "description": "执行 Python 代码",
      "input_schema": {
        "type": "object",
        "properties": {
          "code": {
            "type": "string",
            "description": "要执行的 Python 代码"
          }
        },
        "required": ["code"]
      }
    }
  ]
}
```

### 9.3 流式响应 (Streaming)

```
普通响应: 一次性返回完整内容
┌──────────────────────────────────────────────┐
│ 服务器处理完成 → 返回完整响应                  │

└──────────────────────────────────────────────┘

流式响应: 逐块返回
┌──────────────────────────────────────────────┐
│ 事件: content_block_delta                     │
│ 数据: { "delta": { "text": "这" } }           │
├──────────────────────────────────────────────┤
│ 事件: content_block_delta                     │
│ 数据: { "delta": { "text": "是" } }           │
├──────────────────────────────────────────────┤
│ 事件: content_block_delta                     │
│ 数据: { "delta": { "text": "一" } }           │
├──────────────────────────────────────────────┤
│ ...持续发送...                                │
├──────────────────────────────────────────────┤
│ 事件: message_stop                            │
│ 数据: {}                                      │

```

## 总结与展望

### 核心概念速查表

| 概念 | 一句话定义 | 关键词 |
|------|-----------|--------|
| **LLM** | 预测下一个词的超级模型 | Transformer, Token, Attention |
| **Agent** | 能自主执行任务的 AI | 感知、推理、行动、记忆 |
| **Tool Calling** | 让 LLM 能操作外部世界 | 函数调用, 参数传递, 结果返回 |
| **MCP** | AI 与工具连接的统一协议 | JSON-RPC, Resources, Tools |
| **Skill** | 预打包的能力集合 | 工具 + 文档 + 提示词 |
| **Session** | 对话的生命周期容器 | 消息历史, 状态管理 |
| **OpenClaw** | 开源 Agent 平台 | Gateway, Skills, Nodes |
| **Claude Code** | AI 编程助手 | 代码生成, 项目管理 |
| **Antigravity** | Google 的 AI IDE | OAuth, 模型代理 |

### 技术演进路线

```
2022 ──── 2023 ──── 2024 ──── 2025 ──── 2026
  │         │         │         │         │
ChatGPT   GPT-4    Agent     MCP      Claude Code
  │         │         │         │         │
纯对话   多模态   工具调用   标准化   深度集成
```
